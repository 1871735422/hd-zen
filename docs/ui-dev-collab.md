## UI × 前端协作规范（会议稿）

> 目标：提高还原效率与跨分辨率一致性，减少返工，让设计物能被 AI/工程化工具直接消费。

---

### 一、移动端尺寸建议（两套必出）

- **推荐两套基准尺寸**：
  - **375 × 812（iPhone X/XS/11 Pro 等）**：经典 375 宽度生态最广，适合作为小屏基线。
  - **390 × 844（iPhone 12/13/14/15 标准版）**：覆盖近年主流 iPhone 非 Pro 机型。

- **机型映射（常见）**：
  - 375×812：iPhone X / XS / 11 Pro；部分安卓机在 CSS 像素宽度接近 360～384，实际渲染靠流式与约束适配。
  - 390×844：iPhone 12 / 13 / 14 / 15（标准版）。
  - 393×852：iPhone 14 Pro / 15 Pro（与 390 近似，可通过等比与边界约束兼容）。
  - 360×800（常见安卓）：无需单独出稿，通过流式与断点保证主要布局稳定。

- **执行要点**：
  - 两套尺寸需给出同等完备的页面/组件稿与交互说明。
  - 标注可伸缩/不可伸缩的区域（如卡片网格列数、页边距、按钮最小宽度）。
  - 字体、间距优先引用“设计令牌”（Design Tokens），并在不同尺寸保持一致的比例关系。

---

### 二、桌面端与响应式策略

- **断点建议（至少拍板两档）**：
  - 桌面：**1440**、**1920**（必要时补充 1280 / 1600 / 2560）。
  - 容器化布局：整体用容器 `max-width` 控制行长，内部采用区间缩放（clamp）保证文字与间距在上下限之间流动。

- **缩放规则**：
  - 低于最小断点：固定为最小值，避免过密或断行异常。
  - 介于断点区间：等比/半等比缩放字体与间距（使用 tokens 映射）。
  - 高于最大断点：限制上限，避免“巨屏拉伸”。

---

### 三、UI → 前端：交付清单与规范

1) **设计令牌（Design Tokens）**：
   - 内容：颜色、字号、行高、字重、间距、圆角、阴影、层级、动效时长/曲线。
   - 形式：`tokens.json`（机器可读），命名稳定、英文/拼音小写-kebab-case。
   - 变更：发版节奏与变更日志（哪些 token 变了、为何变）。

2) **组件与变体**：
   - 使用 Auto Layout 与 Constraints、提供尺寸（sm/md/lg）、状态（hover/active/disabled/focus/加载）。
   - 组件命名贴合代码（如 `button/primary/lg`），文案长度边界与溢出规则明确。

3) **页面规格说明**：
   - 断点行为（列数/侧边栏显隐/导航收合）、滚动与空态/错误态、加载骨架、表单校验规则。
   - 图表类注明采样密度、最小展示宽度、Tooltip/缩略交互。

4) **图标规范（仅交付 SVG）**：
   - 画板：统一 24×24；描边粗细（如 1.5px）；端点/拐角统一；对齐像素网格。
   - 命名：`icon/{category}/{name}.svg`（如 `icon/action/close.svg`）。
   - 交付：提供“图标库页面”（集中展示），支持批量导出；禁止 PNG/位图切图。

5) **动效与可用性**：
   - 动效：时长（ms）、缓动曲线、触发条件；焦点可见性与键盘导航范围。
   - 无障碍：颜色对比度标注、可替代文本建议。

---

### 四、前端 → UI：设计可实现性的建议

- **少而稳的刻度体系**：采用 8pt/4pt 间距网格与字体比例尺，避免大量“特殊值”。
- **优先复用组件**：新样式先评估是否为现有组件的“变体”，减少“孤品”。
- **自适应边界清晰**：明确哪些元素可拉伸、最小/最大宽度、文案换行与截断策略。
- **图标统一管线**：仅收 SVG；前端用 SVGO 优化 + Iconify/SVGR 按需引用，禁止在设计里嵌入位图。
- **文案长度与国际化**：提供极端文案示例（长词/多语言），避免布局抖动。
- **表格/卡片密度**：同一页面限制密度变体数量，保证可读性与一致性。

---

### 五、AI 协作与工具选择

- **工具建议**：
  - Figma 生态、插件与 API 更成熟，利于 tokens、组件、SVG 自动化与 AI/CI 流程打通。
  - 若继续用“即时设计”，需补齐：tokens 导出插件、SVG 批量导出、组件变体与约束规范，以及设计→代码的可编程接口。

- **喂给 AI 的结构化资产**：
  - `tokens.json`、组件命名规范、断点行为说明、批量 SVG 图标（统一命名）。
  - 页面/组件的交互说明（文字化），作为生成与审阅时的上下文。

- **自动化（建议 CI 落地）**：
  - 设计仓（或链接）→ CI 生成 `CSS Variables / TS 常量 / Tailwind 主题`。
  - 图标 SVG → SVGO → 自定义 Icon 集（Iconify 或 SVGR）。

---

### 六、会议需拍板的关键项

1) 移动端两套尺寸：**375×812** 与 **390×844**（并明确与 360×800、393×852 的兼容策略）。
2) 桌面断点：至少 **1440** 与 **1920**，并约定容器宽度与缩放边界。
3) 设计令牌标准：命名、文件格式（`tokens.json`）、发版与变更流程。
4) 图标交付与命名：仅 SVG、24×24、描边统一、命名规范与批量导出方式。
5) 组件库规范：变体/状态矩阵、Auto Layout/Constraints 强制使用。
6) AI/CI 流程：设计变更自动生成样式/主题，前后端共享同源 tokens。

### 附录 B：图标命名与交付

- 路径结构：`icon/{category}/{name}.svg`，如 `icon/navigation/chevron-down.svg`。
- 画板：24×24，描边 1.5px，端点与拐角一致，中心对齐像素网格。
- 颜色：默认使用 `currentColor`；避免内联填充固化为具体色值。
- 只交付 SVG，禁止 PNG 位图。
